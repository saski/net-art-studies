# Net Art Critique

## Goal

Help the project develop a coherent visual and conceptual language. Prioritize the artwork's behavior, browser-native qualities, and artistic direction over implementation cleverness.

## Review Order

1. Name what the work is doing as a system.
2. Identify the strongest browser-native material in the piece.
3. Identify anything that feels generic, decorative, or too product-like.
4. Check whether the piece has one clear behavioral question.
5. Suggest one next mutation, not a long feature list.

## Critique Criteria

- Browser behavior is part of the artwork.
- Interaction changes the system meaningfully.
- Determinism supports editioning or reproducible study, but does not flatten the work.
- Visual language connects to stains, crude figures, unstable systems, painterly surfaces, or deliberate imperfection when relevant.
- Documentation explains intent without overexplaining the code.
- Marketplace, minting, and export decisions remain secondary until the work has a strong internal logic.

## Response Shape

Use this structure:

```md
Outcome: [one-sentence assessment]

Strongest signal: [what works]

Weakest signal: [what is diluting the piece]

Next mutation: [one concrete next step]

Checks: [tests or visual inspection notes, if relevant]
```

Keep critique direct, specific, and usable.
